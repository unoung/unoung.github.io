# 🧑‍💻 테크블로그 ( Tech Blog)

Jekyll을 기반으로 구축되었으며, 마크다운(Markdown)으로 글을 작성하고 배포할 수 있습니다.

---

## 🚀 1. 로컬 환경 세팅 및 실행 (Getting Started)

블로그를 내 컴퓨터에서 실행하고 수정 사항을 미리 확인하려면 아래 단계를 따라주세요.
_(💡 필수 환경: Ruby, Bundler, Node.js가 설치되어 있어야 합니다.)_

### 1) 저장소 클론 및 패키지 설치

```bash
# 1. 레포지토리 클론 후
# 2. Jekyll 관련 패키지 설치
bundle install
```

### 2) 로컬 서버 실행

```bash
bundle exec jekyll serve
```

- 서버가 실행되면 브라우저에서 `http://localhost:4000` 으로 접속하여 블로그를 확인할 수 있습니다.
- 마크다운 파일을 수정하고 저장하면 **새로고침 시 즉시 반영**됩니다. (단, `_config.yml`을 수정했다면 서버를 껐다가 다시 켜야 합니다.)

## 📝 2. 새로운 포스트 작성하기 (빠른 생성 스크립트)

날짜를 맞추고 파일과 폴더를 일일이 생성할 필요가 없습니다!
마인드쿡 블로그에는 **글 생성 자동화 스크립트**가 준비되어 있습니다.

터미널을 열고 아래 명령어를 입력해 주세요.

```bash
node make-post.js "여기에 작성할 글 제목을 입력하세요"
# 예시: node make-post.js "zustand 상태 관리 최적화"
```

**✨ 스크립트 실행 시 자동으로 처리되는 작업:**

1. `_posts/2026-MM-DD-zustand-상태-관리-최적화.md` 마크다운 파일 생성 (기본 템플릿 포함)
2. `assets/images/2026-MM-DD-zustand-상태-관리-최적화/` **전용 이미지 폴더 자동 생성**
3. 이제 생성된 마크다운 파일을 열고 바로 본문을 작성하시면 됩니다!

## 🎨 3. 글쓰기 가이드 및 마크다운(Markdown) 사용법

마크다운이 익숙하지 않아도 괜찮습니다. 아래의 규칙만 기억하시면 깔끔하고 예쁜 기술 블로그 글을 완성할 수 있습니다!

### 📌 1) 자동 목차(TOC) 생성 규칙

마인드쿡 블로그는 글의 **제목(Heading)** 태그를 인식하여 최상단에 **자동으로 목차를 생성**합니다.
반드시 `##` (제목 2)와 `###` (제목 3)을 사용해 글의 뼈대를 잡아주세요!

```markdown
## 1. 전환의 계기

알림톡 시스템을 바꾸게 된 이유를 설명합니다.

### 1.1 왜 바뀌야 했을까?

기존 시스템의 문제점은 이랬습니다...

## 2. 전환 방법론

실제 적용한 방법을 소개합니다.
```

_(주의: `#` (제목 1)은 글 전체 제목에 사용되므로 본문에서는 사용하지 않습니다.)_

### 🖼️ 2) 이미지 삽입하기

글에 이미지를 넣을 때는 **`make-post.js`가 만들어준 전용 이미지 폴더**에 사진을 먼저 넣은 뒤, 아래 형식을 사용합니다.

```markdown
![메인 아키텍처 구조](/assets/images/2026-03-01-zustand-상태-관리-최적화/architecture.png)
```

### 💻 3) 코드 블록 작성하기 (문법 하이라이팅 적용)

개발 블로그의 핵심인 코드는 백틱(```) 3개를 사용하여 작성합니다. 첫 줄에 언어 이름(js, ts, java, html 등)을 적으면 예쁘게 색상이 입혀집니다.

```typescript
import { create } from 'zustand';

const useStore = create((set) => ({
  count: 1,
  inc: () => set((state) => ({ count: state.count + 1 })),
}));
```

### ✍️ 4) 자주 쓰는 텍스트 강조 문법

- **굵은 글씨 (Bold)**: `**중요한 내용**`을 별표 두 개로 감쌉니다.
- _기울임 (Italic)_: `*기울임 내용*`을 별표 한 개로 감쌉니다.
- `짧은 코드 (Inline Code)`: 텍스트 중간에 들어가는 코드는 백틱 하나(`코드`)로 감싸면 핑크색 배경으로 예쁘게 강조됩니다.
- 인용구 (Blockquote):
  > 💡 꿀팁이나 남의 글을 인용할 때는 맨 앞에 `>` 를 붙여줍니다.

---

## 🤝 4. 배포

1. 글 작성이 완료되면 로컬 서버(`localhost:4000`)에서 오타나 이미지 엑스박스가 없는지 최종 확인합니다.
2. `feature-post` 브랜치로 -> `git commit -m "post: [포스트제목] 글 발행"` 명령어로 커밋, 푸시 합니다.
3. pr (github action 맞춤법 검사) 후 main에 merge합니다.
